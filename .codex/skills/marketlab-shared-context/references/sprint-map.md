# Sprint Map

## Sprint 1

- Bootstrap the package and CLI.
- Load or reuse cached panel data.
- Engineer trailing, leakage-safe features.
- Run rule baselines.
- Backtest on adjusted daily OHLC.
- Write metrics, plots, and Markdown reports.
- Add unit and integration coverage.

## Sprint 2

- Add target generation.
- Add walk-forward evaluation.
- Add model wrappers.
- Add ranking strategy.

## Sprint 3

- Add Docker.
- Add CI.
- Expand docs and productization.

## Ownership Rule

- If a task can be completed with Sprint 1 files, keep it in Sprint 1.
- If a task requires ML, walk-forward logic, or deployment tooling, mark it as later-sprint work unless the user explicitly asks to expand scope.
